## 포크
원본 저장소의 소유자는 협력자가 많아지면 관리가 어려워짐.
협력자가 원본 저장소에 직접 푸시할 수 있기 떄문..

하지만 많은 개발자에게 의견을 받고, 오픈소스를 개선하고자 하는 욕구가 있음.

이럴 때 포크해서 풀 리퀘스트를 하면 됨.
즉 원본 저장소를 내 계정에 복사해서 커밋을 올린후, 원본 개발자에게 병합요청을 하는거임.


## 남의 저장소를 포크하기
### 이메일 만들기
깃허브에서는 + 기능이 있음.
자기 이메일이 test@test.com 이면 test+1@test.com 이런식으로 계정을 만들어도 test@test.com에 메일이 옴.

### 포크하기
깃허브 사이트에 들어가서 포크 버튼을 누르면 됨.
만약 main 브랜치만 가져오고 싶으면, Copy the main branch only 를 체크하면 됨.

그리고 소스트리에서 클론을 하면 됨.
그리고 설정에서 2번째 계정을 추가.

### 커밋
2번쨰 계정으로 like.md 를 만들어서 "좋아요 기능" 작성.
그리고 커밋하고, 푸시.

## 원본 저장소에 풀 리퀘스트 보내고 병합
### 포크한 저장소에서 원본 저장소로 리퀘스트 보내기
- 서브계정으로 로그인하고 깃허브 레포 접속
- Contribute 누르고 open pull request 클릭
- 들어가서 Able to merge 가 보이면 충돌없이 병합 가능하다는 뜻
- pr 설명을 쓰고 create pull request 하기

PR 컨트리뷰션 가이드 -> [바로가기](https://legacy.reactjs.org/docs/how-to-contribute.html)

### 풀 리퀘스트 승인
- 메인계정으로 로그인하고 깃허브 레포 접속
- insights -> forks에 가면 누가 포크했는지 나옴
- pull requests로 가면 PR 목록이 나옴.
- Merge pull request 클릭
- Confirm merge 클릭

## 리베이스
### 원본 저장소에 새로운 커밋이 있는데, 포크한 내 원격 저장소(서브 저장소)에는 안보여요!!
- vsc로 서브 레포에 들어가기
- like.md에 찜하기 기능 추가
- 커밋후 푸시

- 소스트리 메인 레포 들어가기
- 왼쪽 main에 우클릭
- Pull origin/main (tracked) 클릭

이제 충돌을 일으킬거임

- 메인레포 like.md에 싫어요 기능 추가
- 커밋하고 푸시
- 메인 레포 README.txt에 3. 너구리 추가
- 커밋하고 푸시
- 서브계정으로 깃허브 들어가서 open PR 클릭

그러면 Can`t automatically merge 이라고 뜸.<br>
그래서 충돌을 해결하고 PR을 보내야함.

### 리모트 추가
- 상단 설정 클릭
- 원격 클릭
- 원격 이름을 upstream으로 해놓고, url은 원본 저장소 주소
- 왼쪽 upstream 브랜치를 우클릭, upstream에서 가져오기

### 리베이스하기
- 개발자 목록에 너구리 추가를 우클릭, 재배치 클릭
- 서브저장소에 vsc로 들어가서 like.md 수정
