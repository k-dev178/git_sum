## 어멘드
### 로컬 저장소의 맞막 커밋 수정하기
만약 커밋을 했는데 뒤늦게 파일이 빠진걸 알았음.

그럴땐 어멘드를 사용하면 됨.
어멘드는 방금한 커밋을 수정하는 기능임.

1. amend.md 생성후, "어멘드 실습하기" 작성
2. 어멘드 실습파일 추가 라고 작성후 커밋

<img width="1226" height="157" alt="스크린샷 2026-04-02 오후 3 27 46" src="https://github.com/user-attachments/assets/b3b98449-a758-4899-b78b-d35607973185" />

4. vsc에서 "어멘드 실습하기"를 "amend 실습하기"로 수정
3. 마지막 커밋 수정 클릭
4. 커밋 클릭

### 원격 저장소의 마지막 커밋 수정하고 강제 푸시하기
이제는 푸시한거도 수정하는 방법이 나옴.

1. 방금한거 푸시
2. 마지막 커밋 정정 누르고, "커밋 메시지 수정"추가 작성후 다시 커밋
3. 강제푸시 켜고, 푸시

## 체리픽
### 브렌치 전략 살펴보기

| 브랜치명 | 특징 |
|:---:|:---|
| feat/기능이름 | <div align="left">* 각 개발자가 개발중인 브랜치<br>* 직접 커밋을 올림</div> |
| main | <div align="left">* [feat/기능이름]브랜치에서 개발완료한 코드가 합쳐긴 브랜치<br>* 출시 전의 베타 버전<br>* 직접 커밋을 올리지 않음(병합을 통해서만 코드를 업데이트)</div> |
| latest | <div align="left">* 실제 출시할 코드(대중에게 보여줄 완별한 코드)를 올리는 브랜치<br>* [main]브랜치에서 굵직한 개발이 끝나면 출시 시점에 [latest] 브랜치로 코드를 병합.</div> |

내가 병합하기 원하는 커밋(체리)만 선별해서 병합하는 것을 체리 픽이라고 함.
메인브랜치에서 개발을 하다가, 기존 서비스의 버그를 발견함. 그래서 그 메인 브랜치에서 따로 fix 브랜치를 만들고 버그를 수정.

그 수정된 코드만 latest 파일에 적용하게 해줌.


### 다른 브랜치의 커밋 하나만 내 브랜치에 반영하기
1. feat/a 브랜치 만들기
2. README.md 에 "깃으로 놀아보자!" 추가
3. 커밋, 메시지는 "첫 번째 커밋"

1. cherrypick.md 파일 만들고, "체리 픽 실습하기" 텍스트 작성
2. 커밋, 메시지는 "두 번째 - 커밋 - 탐나는 커밋"

1. README.md에 "Let's git it!" 추가
2. 커밋, 메시지는 "세 번째 커밋"
